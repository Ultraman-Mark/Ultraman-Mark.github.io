Spring Cloud-项目-业务思想：

优惠券模板微服务

优惠券模板创建：

优惠券微服务，优惠券模板分离：

  • 方便创建，一次创建，多次使用

  • 二者使用人群不同：优惠券主要为用户使用，优惠券模板主要为运营人员使用

给优惠券模板生成”优惠券码“并保存到redis

优惠券码先在服务实例中预先生成，并放到Redis中，目的：

  • 简单的解决一致性问题（Set数据结构）
  • 不会造成优惠券码超发
  • 不需要重复考虑分发优惠券的配额问题

不好的实现

好的实现

清理过期的优惠券模板

策略一：template模块自己定期清理策略（定时任务）

策略二：使用template模块的其他模块自己校验（策略一存在延迟）

优惠券分发微服务

根据用户id和优惠券状态查找用户优惠券

说明：

  1. 由于不涉及用户系统，用户id不做校验，需要fake
  2. 优惠券状态有三类：可用，已使用，过期（未使用）
  3. 用户数据都存储在Redis中
  4. 除获取用户的优惠券外，还有延迟的过期处理策略

耗时操作：

1.检索用户优惠券（缓存解决）

2.处理过期优惠券（Kafka异步处理）

根据用户id查找当前可以领取的优惠券模板

说明：

  • 优惠券模板从template模块处获取（熔断兜底策略）
  • 根据优惠券的领取限制，比对当前用户所拥有的优惠券，做出判断

用户领取优惠券

说明：

  1. 优惠券模板从template模块处获取（熔断兜底策略）
  2. 根据优惠券的领取限制，比对当前用户所拥有的优惠券，判断是否可以领取
  3. 从Redis中获取优惠券码
  4. 优惠券写入MySQL和Redis中

结算（核销）优惠券

说明：

  1. 校验需要结算的优惠券是否是”合法的（属于用户&&可用）“
  2. 利用settlement模块计算结算数据
  3. 如果是核销，需要写进数据库

核销和结算不是同一个操作

结算微服务

根据优惠券类型结算优惠券

  • 优惠券分类，不同的优惠券有不同的方法
  • 不同的优惠券有些可以组合，有些不可以组合

优惠券系统存储设计

优惠券模板表：coupon_template

Field Type Null Key Default
id int(11) no PRI 主键
available tinyint(1) no 可用状态
expired tinyint(1) no 是否过期
name varchar(64) no UNI 名称
logo varchar(256) no logo
intro varchar(256) no 描述
category varchar(64) no MUL 分类
product_line int(11) no 生产线
coupon_count int(11) no 个数
create_time datetime no 创建时间
user_id bigint(20) no MUL 创建模板运营人员id
template_key varchar(128) no 模板描述
target int(11) no 扩展
rule varchar(1024) no 优惠券规则

说明

  • name是唯一字段，id是主键
  • category和user_id建立了单列索引
  • rule是为了方便存储为json字符串

（用户）优惠券表：coupon

Field Type Null Key Default
id int(11) NO PRI 主键
template_id int(11) NO MUL 优惠券模板外
user_id bigint(20) NO MUL 用户id
coupon_code varchar(64) NO 优惠券码
assign_time datetime NO 领取时间
status int(11) NO 优惠券三类状态

template_id和user_id建立了单列索引

优惠券码缓存

说明：

  1. 优惠券系统使用Redis做缓存
  2. Key要有意义，且不能与原有的Key冲突
  3. 优惠券码需要永远存在，不设过期时间

  • Key = 前缀（immoc_coupon_template_code_）+后缀(优惠券模板数据表主键)

  • Value类型：list

优惠券码（用户）缓存

说明：

  1. 优惠卷系统使用Redis做缓存
  2. Key要有意义，且不能与原有的Key冲突
  3. 用户优惠券信息根据状态(未使用，已使用，已过期)分为三类
  4. 用户数据存储在MySQL中，数据量大，不适合长期存储在redis中，需要设计过期时间

  • Key = 前缀（immoc_user_coupon__usable、imooc_user_coupon_used、imooc_user_coupon_expired）+后缀(优惠券模板数据表主键)

  • Value类型：hash(key:优惠券id；value:优惠券信息)

给定一个非空字符串 s 和一个包含非空单词的列表 wordDict，判定 s 是否可以被空格拆分为一个或多个在字典中出现的单词。

说明：

  • 拆分时可以重复使用字典中的单词。
  • 你可以假设字典中没有重复的单词

示例 1：

输入: s = “leetcode”, wordDict = [“leet”, “code”]
输出: true
解释: 返回 true 因为 “leetcode” 可以被拆分成 “leet code”。

示例 2：

输入: s = “applepenapple”, wordDict = [“apple”, “pen”]
输出: true
解释: 返回 true 因为 “applepenapple” 可以被拆分成 “apple pen apple”。
注意你可以重复使用字典中的单词。

示例 3：

输入: s = “catsandog”, wordDict = [“cats”, “dog”, “sand”, “and”, “cat”]
输出: false
